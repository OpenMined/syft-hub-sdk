Architecture Overview

The SDK provides a Client for connecting to your SyftBox, discovering AI services offered by peers, and composing a federated RAG pipeline that tracks attribution.

Client

  • Manages connection to SyftBox and caches
  • Handles authentication lifecycle and session state
  • Provides discovery, health checks, and cost monitoring

AI Services

Services expose capabilities on the network and run on peers:

  • /chat: text generation with parameters like temperature and max_tokens
  • /search: semantic retrieval with corpus selection and ranking parameters

Service Discovery

Use indexes and filters to find services by type, owner, health, and attributes. Service metadata includes capabilities and attribution support.

Federated Pipeline

Compose multiple data sources with a synthesizer model to run federated RAG queries. The pipeline orchestrates retrieval, synthesis, and attribution.

Attribution

Attribution-based control ensures every query goes to its data source, enabling transparent provenance and contribution tracking across sources and models.
